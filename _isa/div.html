{{ page.title }}

Group: arithmetic

{{ page.summary }} Both operand numbers can be of different types, result is determined by result type specifier.

Syntax

{% highlight text %} (0) div return-type-specifier r0 r1 r2 (1) div return-type-specifier r0 r1 {% endhighlight %}

(0) is the canonical form. Assembler expands (1) to "div return-type-specifier r0 r0 r1".

{% highlight lua %} div int8 %1 local %2 local -- expands to... div int8 %1 local %1 local %2 local {% endhighlight %}

Operands "r1" and "r2" may use pointer dereference.

"return-type-specifier" may be one of: int8, int16, int32, int64, float32, float64.

Semantics

Divides number in "r1" by number in "r2" and stores the result in "r0". Type of the result of said operation is determined by the "return-type-specifier". Input operands are signed integers or floating point numbers.

Examples

{% highlight lua %} .function: example/0 istore %1 local 126 -- store 3 in local register 1 istore %2 local 3 -- result will be stored in "%3 local" -- its type will be a 64 bit signed integer div int64 %3 local %1 local %2 local -- prints "42" to stdout print %3 local return .end {% endhighlight %}
