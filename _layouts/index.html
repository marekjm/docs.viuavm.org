Getting started with Viua VM is straightforward; it is the standard "clone-make-test" procedure. Executing the follwing commands should give you a fully functioning VM (remeber to use GCC 5, GCC 6 or recent version of Clang++; the compiler you use must support at least C++11):

{% highlight bash %} # create an anonymous, read-only clone $ git clone --recursive --branch devel {{ site.savannah_anonymous_clone_url }} $ cd viuavm $ vi Makefile # set CXX_STANDARD flag to c++11 or c++14 (recommended), depending on # what your compiler supports $ make # compile the code $ ./scripts/test_quick # perform quick test to see if everything works as it should {% endhighlight %}

For more information see installation instructions.

Alpha software

Viua VM is alpha software, and is undergoing constant change. The scope of code-breaking changes is kept under control on a best-effort basis, if something needs breaking it is being broken; privileges of alpha.

You can get the newest code on Savannah or Github.

Manuals

Manuals are articles of varying length, explaining how to use various features of the machine.

Contents

Manuals contain both introductory and advanced material, in no particular order. They explain the basics ("Hello World!", syntax of VMs assembly, etc.) and more arcane (concurrency primitives, resource management, FFI modules, etc.) features of the VM.

Intended audience

All articles assume that the reader has prior programming experience, and is reasonably proficient in the craft. In-depth articles assume prior C++ and Viua VM programming experience.

References

References also contain VM instruction set reference, and bytecode format description.

References are lists of features, with short examples of usage (in the spirit of cppreference.com). They are intended either for people who just need to refresh their memory (what arguments does a function take, in which module was a function implemented, etc.), or people who are interested in feature lists.
